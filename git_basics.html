Git と GitHub の基本
📅 作成日: 2023年5月1日

📚 Gitとは

Gitはバージョン管理システムです。簡単に言えば「タイムマシン付きのノート」で、プログラムの変更履歴を記録し、いつでも過去の状態に戻れるようにします。

📝 Gitはプログラマーの時間旅行機械！

🔄 基本的なワークフロー

  1. git init
    新しいリポジトリを初期化します
    ✅ プロジェクトを始めるとき、一度だけ使います
  2. git add .
    変更をステージングエリアに追加します
    ✅ . はすべての変更を追加する意味です
  3. git commit -m "メッセージ"
    ステージングエリアの変更を記録します
    ✅ メッセージには何を変更したかを書きます
  4. git status
    現在の状態を確認します
    ✅ 変更されたファイル、ステージング状況を表示

🌐 リモートリポジトリ操作

  1. git remote add origin <URL>
    リモートリポジトリを追加します
    ✅ GitHubのリポジトリと連携するときに使います
  2. git push
    ローカルの変更をリモートに送信します
    ✅ 初回は git push -u origin main と指定
  3. git pull
    リモートの変更をローカルに取り込みます
    ✅ 共同作業するときに使います

☁️ GitHubとは

GitHubはGitリポジトリのホスティングサービスです。「冒険ノートを保管する雲の上の図書館」のようなものです。

💡 GitHubはコードのSNS！

✨ GitHubの主な利点

  • バックアップ: コードをクラウドに保存
  • 共有: チームでの開発を容易にする
  • 公開: オープンソースプロジェクトへの貢献
  • ポートフォリオ: 自分のコードを公開できる

📋 Git/GitHubの基本的な使い方

  1. ローカルでファイルを編集する
  2. git add . で変更をステージングに追加
  3. git commit -m "変更内容" で変更を記録
  4. git push でGitHubに変更をアップロード
